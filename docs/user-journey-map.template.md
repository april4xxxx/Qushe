### 无障碍检查（GoogleUX 6.6）

- **触控**: 所有按钮 >= 44pt？单手可达？
- **视觉**: 字号合适？对比度足够？支持屏幕阅读器？
- **听觉**: 有音频内容时有字幕/文字替代？
- **语音**: 不能说话的用户有替代输入？

### 设计偏见检查（GoogleUX 6.7）

- Journey 是否基于真实用户测试，而非设计师自我想象？
- 是否有多个 Persona 覆盖不同特征？
- 是否避免了强化刻板印象？

---

## 流程规则

1. **任何新功能动手前**必须先画对应 Journey Map（至少 5 个 Action + 情绪 + 痛点）
2. **每次大版本测试后**回来更新已有 Journey 的"状态"列
3. **痛点优先级**用 ICE 框架排序（见 `design.template.md` 第六章）
